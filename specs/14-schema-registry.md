# Design — Schema Registry & Sink Decode Contract

Status: draft v1 · Owner: obs-core · Last updated: 2026-05-02 · Depends on: [11-runtime-core.md](./11-runtime-core.md), [12-schema-and-codegen.md](./12-schema-and-codegen.md)

This spec defines the runtime *schema registry* — the in-process,
link-time-collected map from `(schema_hash, full_name)` to an
object-safe `EventSchemaErased` view of every `EventSchema` in the
binary. It is the contract that lets sinks (`OtlpLogSink`,
`ParquetSink`, `ClickHouseSink`, the bridge's `ObsToTracingSink`)
**decode the typed payload** without depending on the user's codegen
crate.

Earlier drafts referenced `inventory::collect!` in passing
([13-emit-scope-and-filter.md § 2.3](./13-emit-scope-and-filter.md#23-validation-of-declared-fields))
but never specified the contract; this spec closes that gap. The
sink chapter ([20-otel-and-sinks.md § 3.1](./20-otel-and-sinks.md#31-the-sink-trait))
assumes the registry exists.

> v1 (this draft): pulled out of the codegen + sink specs into a
> standalone surface; picks `linkme` over `inventory` for the link-
> section primitive (rationale in § 7); introduces `ScrubbedEnvelope`
> as the type-system handoff between the worker scrubber and the
> sink chain.

## 1. The problem

`EventSchema` is **not object-safe** — it has associated `const`s
(`FULL_NAME`, `TIER`, `DEFAULT_SEV`, `FIELDS`, `SCHEMA_HASH`) and a
generic `Sized` bound from `encode_payload(&mut BytesMut)`. So sinks
cannot hold `&dyn EventSchema`.

But sinks need to:

- **OTLP**: turn `env.payload` bytes into a `KeyValueList` body and
  per-`MEASUREMENT`-field metric data points.
- **Parquet/ClickHouse**: turn `env.payload` bytes into a typed
  Arrow `StructArray` row, populating the right `payload_<full_name_snake>`
  column.
- **Bridge (Direction B)**: synthesise a `tracing::Metadata` whose
  `FieldSet` matches the schema's fields.
- **CLI `obs decode`**: render `env.payload` as JSON given only
  `full_name`.
- **Scrubber**: redact PII / strip SECRET fields per the schema's
  classification annotations ([70-security-and-classification.md § 4](./70-security-and-classification.md#4-the-payload-scrubber)).

Each sink does this differently, but all of them need the same
primitive: *an object-safe view of one schema, looked up by name or
hash, walkable by codegen-generated dispatch tables.*

## 2. The `EventSchemaErased` trait

```rust
/// Object-safe complement to `EventSchema`. Generated by the codegen
/// for every `Obs*` event type and registered into a link-time
/// distributed slice (§ 3). Sinks consume `&'static dyn
/// EventSchemaErased` looked up via the registry (§ 4).
///
/// All methods are `&self`; per-call mutable scratch is passed in.
pub trait EventSchemaErased: Send + Sync + 'static {
    /// Stable identity (matches `EventSchema::FULL_NAME`).
    fn full_name(&self) -> &'static str;

    /// First 8 bytes of BLAKE3 over the canonical descriptor; baked
    /// at build time. Matches `EventSchema::SCHEMA_HASH`.
    fn schema_hash(&self) -> u64;

    /// Tier and default severity for routing decisions.
    fn tier(&self) -> Tier;
    fn default_sev(&self) -> Severity;

    /// Field metadata table; same memory as `EventSchema::FIELDS`.
    fn fields(&self) -> &'static [FieldMeta];

    /// Decode the buffa-encoded payload and emit metric data points
    /// for every `FIELD_KIND_MEASUREMENT` field. Used by metric sinks.
    /// Implementations call `EventSchema::project_metrics` after a
    /// zero-copy view decode.
    fn project_metrics(&self, payload: &[u8], emitter: &mut dyn MetricEmitter)
        -> Result<(), DecodeError>;

    /// Decode the payload into a `StructArray` row whose schema matches
    /// the codegen-emitted Arrow fragment for this event type. Used by
    /// `ParquetSink` and `ClickHouseSink`.
    fn decode_to_arrow_struct(
        &self,
        payload: &[u8],
        builder: &mut dyn ArrowStructBuilder,
    ) -> Result<(), DecodeError>;

    /// Decode the payload into a flat `KeyValueList` body for OTLP
    /// `LogRecord.body`. Used by `OtlpLogSink` when `decode_body=true`.
    fn decode_to_otlp_kv(
        &self,
        payload: &[u8],
        out: &mut Vec<(&'static str, OtlpValue)>,
    ) -> Result<(), DecodeError>;

    /// Render the payload as a JSON object value (no envelope). Used
    /// by `obs decode`, `obs tail` formatters, and `StdoutSink`'s
    /// `FormatterStyle::Json`.
    fn render_json(
        &self,
        payload: &[u8],
        out: &mut serde_json::Map<String, serde_json::Value>,
    ) -> Result<(), DecodeError>;

    /// Strip / redact classified fields **in place**. Returns the
    /// re-encoded payload bytes (in a thread-local scratch buffer
    /// owned by the worker; cleared between events).
    ///
    /// The scrubber runs in the per-tier worker, between the sampler
    /// and the sink chain (see [11-runtime-core.md § 4](./11-runtime-core.md#4-per-tier-workers-and-sinks)
    /// and [70-security-and-classification.md § 4](./70-security-and-classification.md#4-the-payload-scrubber)).
    fn scrub_for_log(
        &self,
        payload: &[u8],
        scratch: &mut bytes::BytesMut,
    ) -> Result<&[u8], ScrubError>;

    /// Returns the codegen-derived OTel attribute set for the per-event
    /// `event.name` plus any per-event constant attributes. Cheap; no
    /// allocation.
    fn otel_attribute_view(&self) -> &'static OtelAttributeView;
}

#[derive(Debug, thiserror::Error)]
pub enum DecodeError {
    #[error("payload truncated at offset {0}")]
    Truncated(usize),
    #[error("unknown wire-tag {0} (preserve_unknown_fields=false)")]
    UnknownTag(u32),
    #[error("invariant violated: {0}")]
    Invariant(&'static str),
}

#[derive(Debug, thiserror::Error)]
pub enum ScrubError {
    #[error("payload re-encode failed at field {0}")]
    ReencodeFailed(&'static str),
}
```

The codegen (`obs-build` + `obs-macros::derive(Event)`) emits the
`EventSchemaErased` impl alongside `EventSchema`. They share the same
underlying buffa decoder — `decode_to_arrow_struct` and friends are
thin walks over the payload bytes that dispatch into per-field code
generated from the descriptor.

## 3. Link-time registration: `linkme` over `inventory`

The codegen registers each `&'static dyn EventSchemaErased` into a
**link-time distributed slice** so the runtime can enumerate every
schema in the binary without a startup callback dance.

We pick **`linkme`** over `inventory`:

| | `linkme` | `inventory` |
| --- | --- | --- |
| Compile-time enumeration | yes (linker section) | yes (constructors) |
| Works on musl-static | yes (link sections preserved) | partial (init-arrays may be stripped) |
| Works on macOS / Windows | yes | yes |
| Works on WASM | yes | partial |
| Compile-time error on duplicates | yes (link error) | no (silent) |
| Startup cost | zero (slice exists at link time) | one-time `ctor`-style init walk |
| Predictability under `cargo install --no-default-features` | high | known footguns with feature-gated `inventory::submit!` |

The "compile-time error on duplicates" property is the deciding
factor: two crates in the workspace defining the same `full_name`
produce a link error, which is the right failure mode (the alternative
is a silent last-write-wins at startup).

```rust
// In `obs-core::registry`:
use linkme::distributed_slice;

#[distributed_slice]
pub static EVENT_SCHEMAS: [&'static dyn EventSchemaErased] = [..];

// Codegen emits, per event type:
#[linkme::distributed_slice(obs_core::registry::EVENT_SCHEMAS)]
#[linkme(crate = obs_core::__private::linkme)]
static __SCHEMA_OBS_REQUEST_COMPLETED: &dyn EventSchemaErased =
    &ObsRequestCompletedSchema;
```

`linkme` is re-exported from `obs-core::__private` so user crates do
not need to depend on it directly.

## 4. The runtime registry

```rust
pub struct SchemaRegistry {
    /// O(1) lookup by full_name. Built once at observer init from
    /// EVENT_SCHEMAS.
    by_name: HashMap<&'static str, &'static dyn EventSchemaErased>,
    /// O(1) lookup by schema_hash (envelope-side; cheaper than name).
    by_hash: HashMap<u64, &'static dyn EventSchemaErased>,
    /// Combined Arrow schema for all registered events; built once.
    arrow_schema: Arc<arrow_schema::Schema>,
}

impl SchemaRegistry {
    /// Built at `StandardObserver::build()` time. Walks
    /// `EVENT_SCHEMAS`; populates both maps; assembles the unified
    /// Arrow schema.
    pub fn from_link_section() -> Self;

    /// Hot-path lookup. Sinks call this once per envelope.
    pub fn lookup(&self, env: &ObsEnvelope) -> Option<&'static dyn EventSchemaErased> {
        // Try schema_hash first (8 bytes, no string compare).
        self.by_hash.get(&env.schema_hash).copied()
            // Fall back to full_name for unknown-hash interop.
            .or_else(|| self.by_name.get(env.full_name.as_str()).copied())
    }

    /// Combined Arrow schema for the unified `obs_events` table.
    /// Used by `ParquetSink::new` and `ClickHouseSink::auto_migrate`.
    pub fn arrow_schema(&self) -> &Arc<arrow_schema::Schema>;

    /// All registered event types; used by `obs migrate clickhouse`,
    /// `obs schema show`, the bridge's pre-warm path
    /// ([31-callsite-interning.md § 3.3](./31-callsite-interning.md#33-registration-flow)).
    pub fn iter(&self) -> impl Iterator<Item = &'static dyn EventSchemaErased>;
}
```

The registry is owned by `StandardObserver`. Sinks receive
`Arc<SchemaRegistry>` at construction (`Sink::with_registry(...)` is a
factory method) so they can `lookup()` per envelope without a global.

### 4.1 Lookup ordering: hash before name

`schema_hash` lookup is one `u64` hash, no string comparison; about
10–15 ns. `full_name` lookup hashes a `&str` and compares it; about
40–60 ns for typical names. Two-level lookup is cheaper than a
single-name lookup in 99% of cases since envelopes carry both.

### 4.2 What if the lookup misses?

Three causes in practice:

1. **Unknown `full_name` from a foreign producer** — e.g. `obs decode`
   reads a binary `ObsBatch` from another service. Sinks fall back to
   the *unknown-schema renderer*: the OTLP body becomes `bytes(payload)`,
   the Arrow row goes into the `payload_proto: bytes` column, the JSON
   renderer prints `{"_unknown_schema": true, "full_name": ..., "raw_b64": ...}`.
2. **Workspace bug**: a producer crate registered `Foo`, a consumer crate
   uses `Foo` in a different version. Linker would catch identical full_name;
   *similar* full_name is a manual error and `obs lint` flags it.
3. **Bridged interned envelope without a matching `ObsCallsiteRegistered`** —
   already handled by [31-callsite-interning.md § 5.3](./31-callsite-interning.md#53-late-registry-on-the-consumer).

For each miss, increment `obs.runtime.v1.ObsSchemaUnknown` (already in
the self-events catalogue, [11-runtime-core.md § 10](./11-runtime-core.md#10-self-events)),
rate-limited per `(sink, full_name)` to avoid log floods.

## 5. The `ScrubbedEnvelope` worker handoff

Earlier drafts had `Sink::deliver(&self, env: &ObsEnvelope)`, but the
payload scrubber must mutate the envelope's payload bytes between
the sampler and the sinks ([70-security-and-classification.md § 4](./70-security-and-classification.md#4-the-payload-scrubber)).
We encode the scrubbed-or-not state in the type system:

```rust
/// Read-only view of an envelope whose payload has been processed
/// by the per-tier worker's scrubber. Sinks see this; they cannot
/// see the raw envelope. Workers construct it via
/// `ScrubbedEnvelope::scrub(...)`.
pub struct ScrubbedEnvelope<'a> {
    inner: &'a ObsEnvelope,
    payload: &'a [u8],          // either env.payload or the scratch slice
    schema:  Option<&'static dyn EventSchemaErased>,
}

impl<'a> ScrubbedEnvelope<'a> {
    /// Worker-side: run the scrubber, build the wrapper.
    pub(crate) fn scrub(
        env: &'a ObsEnvelope,
        registry: &SchemaRegistry,
        scratch: &'a mut bytes::BytesMut,
    ) -> Result<Self, ScrubError>;

    pub fn envelope(&self) -> &ObsEnvelope;
    pub fn payload(&self) -> &[u8];
    pub fn schema(&self) -> Option<&'static dyn EventSchemaErased>;
}

pub trait Sink: Send + Sync + 'static {
    fn deliver(&self, env: ScrubbedEnvelope<'_>);
    fn flush(&self) -> Pin<Box<dyn Future<Output = ()> + Send + '_>>;
    fn shutdown(&self) -> Pin<Box<dyn Future<Output = ()> + Send + '_>>;
}
```

This is the **corrected** `Sink` trait; supersedes the version in
[11-runtime-core.md § 4](./11-runtime-core.md#4-per-tier-workers-and-sinks)
(which lists `&ObsEnvelope` directly). Cross-reference will be added
to that spec.

Ergonomics: a sink that does not care about the typed payload (e.g.
the `MetricSink` writing only `env.labels`) ignores
`env.schema()` and `env.payload()`. A sink that needs the typed
payload calls `env.schema()?.decode_to_arrow_struct(env.payload(), ...)`.

The `'a` lifetime ties the scrubbed payload to the worker's scratch
buffer, so the sink cannot escape a reference past the per-event
boundary. This matches the existing zero-allocation invariant.

## 6. Init-time vs steady-state work

| Phase | Work | When |
| --- | --- | --- |
| Codegen (compile-time) | Emit `EventSchemaErased` impl + `#[distributed_slice]` registration | `cargo build` |
| Link (compile-time) | Linker assembles `EVENT_SCHEMAS` slice | `cargo build` final stage |
| Observer init (runtime) | Walk `EVENT_SCHEMAS`; build `by_name` / `by_hash`; assemble Arrow schema; emit `ObsRegistryInitialized` self-event | Once per `install_observer` call |
| Steady state (runtime) | `registry.lookup(env)` → 1 HashMap probe | Per envelope, per sink that needs decode |

Init cost is bounded by the number of distinct event schemas
(typically ≤ 10³ in pathological workspaces, ≤ 10² in realistic ones).
Wall-clock budget: ≤ 10 ms for 1000 schemas on M2-class hardware,
dominated by the Arrow schema assembly. Validated by
`bench_registry_init` in `crates/obs-core/benches/`.

## 7. Codegen contract (what `obs-build` emits)

For each event type `ObsXxx` in `myapp.v1`:

```rust
// Generated in $OUT_DIR/obs/registry.rs:

pub struct ObsXxxSchema;

impl EventSchemaErased for ObsXxxSchema {
    fn full_name(&self) -> &'static str { ObsXxx::FULL_NAME }
    fn schema_hash(&self) -> u64        { ObsXxx::SCHEMA_HASH }
    fn tier(&self) -> Tier              { ObsXxx::TIER }
    fn default_sev(&self) -> Severity   { ObsXxx::DEFAULT_SEV }
    fn fields(&self) -> &'static [FieldMeta] { ObsXxx::FIELDS }

    fn project_metrics(&self, payload: &[u8], emitter: &mut dyn MetricEmitter)
        -> Result<(), DecodeError>
    {
        let view = ObsXxxView::from_bytes(payload)?;
        // codegen-emitted dispatch: one record_* per MEASUREMENT field
        emitter.with_attributes(view.label_attrs());
        emitter.record_histogram("myapp.v1.obs_xxx.latency_ms",
            view.latency_ms() as f64, Some("ms"), &[1.0, 5.0, ...]);
        Ok(())
    }

    fn decode_to_arrow_struct(...) -> ... { /* generated */ }
    fn decode_to_otlp_kv(...)    -> ... { /* generated */ }
    fn render_json(...)          -> ... { /* generated */ }
    fn scrub_for_log(...)        -> ... { /* generated; calls per-classification helpers */ }
    fn otel_attribute_view(&self) -> &'static OtelAttributeView { &OBS_XXX_OTEL_VIEW }
}

#[linkme::distributed_slice(obs_core::registry::EVENT_SCHEMAS)]
#[linkme(crate = obs_core::__private::linkme)]
static __SCHEMA_OBS_XXX: &dyn EventSchemaErased = &ObsXxxSchema;
```

`obs::include_schemas!("myapp.v1")` includes this file alongside the
existing codegen artefacts ([12-schema-and-codegen.md § 3.1](./12-schema-and-codegen.md#31-generated-artifacts-per-proto-file));
no extra user-side wiring.

## 8. Failure modes

| Failure | Behaviour |
| --- | --- |
| Two crates register the same `full_name` (e.g. `myapp.v1.ObsXxx`) | **Link-time error** from `linkme` ("duplicate symbol in distributed slice"). Best failure mode possible. |
| Two crates register the same `schema_hash` but different `full_name` (BLAKE3 collision) | Detected at observer-init walk; emit `ObsCallsiteHashCollision` (already in [11-runtime-core.md § 10](./11-runtime-core.md#10-self-events)); registry retains both, hash lookup picks the one with matching `full_name`. |
| Sink looks up an envelope whose schema is not registered | Sink falls back to raw-payload mode (§ 4.2); rate-limited `ObsSchemaUnknown` self-event. |
| `decode_to_arrow_struct` fails (truncated or invalid payload) | Sink returns the row with NULL payload column; emits `obs.runtime.v1.ObsSinkFailed{reason=decode_error}` (rate-limited per `(sink, full_name)`). |
| Scrubber fails to re-encode a payload | Worker drops the envelope; emits `ObsSinkFailed{reason=scrub_error}`. The unscrubbed envelope is **never** delivered to a sink. |
| `EVENT_SCHEMAS` is empty (no codegen ran, or stripped by aggressive LTO) | `StandardObserver::build()` returns an error suggesting the user verify `obs::include_schemas!` was called and `linkme` wasn't stripped. |

## 9. WASM and `no_std`

`linkme` works on WASM in single-binary mode; `no_std` users do not
get the registry at all (no link sections). The `obs-types` crate
remains `no_std`-clean as before; the registry lives in `obs-core`
which is `std`-only by design. WASM is a roadmap item, not v1.

## 10. CLI consumption

| Command | Uses registry for |
| --- | --- |
| `obs lint` | Cross-event uniqueness checks (L013) |
| `obs schema show <full_name>` | Look up the `EventSchemaErased`; render its `fields()`, `tier()`, etc. |
| `obs decode` | Resolve `env.full_name` → `EventSchemaErased::render_json` |
| `obs migrate clickhouse` | Iterate registry; emit one `Nested(...)` column per schema |
| `obs migrate parquet` | Take the unified `arrow_schema()` |
| `obs query --grep` | `render_json` per envelope before substring match |

The CLI links the same way user binaries do: `cargo install obs-cli`
links a registry that contains only the built-in schemas
(`obs.v1.*`). To inspect a user binary's events, the user binary's
crate is added as a dep of `obs-cli` (rare; usually `obs decode` is
fed an FDS via `--schemas proto/`).

## 11. Key Design Decisions

### KD1 — `linkme` over `inventory`

Compile-time error on duplicates. Reliable on musl-static and WASM.
Zero startup walk. Two-line codegen change to switch back if upstream
breaks.

### KD2 — Object-safe trait, not codegen-emitted dispatch tables

A `match env.full_name { ... }` in every sink would put the dispatch
table in user code instead of in the generic sink. The trait moves it
to the schema, which the codegen owns. Sink crates stay user-code-free.

### KD3 — `ScrubbedEnvelope` is the type-system handoff

`Sink::deliver` cannot accidentally see an unscrubbed payload; the
worker owns the scratch buffer; lifetime is bounded to the per-event
call. Replaces the implicit "the worker scrubs first" rule that was
buried in [11-runtime-core.md § 4](./11-runtime-core.md#4-per-tier-workers-and-sinks).

### KD4 — Lookup tries `schema_hash` first

8-byte u64 lookup is faster than string lookup; envelopes carry both;
fall-through to `full_name` covers the foreign-producer case.

### KD5 — Init-time Arrow schema, not per-write

The unified Arrow schema for the `obs_events` table is built once at
observer init from the registry, not per Parquet file or per
ClickHouse INSERT. Saves O(N×files) work; matches what
[22-analytics-storage.md § 1.1](./22-analytics-storage.md#11-table-shape)
implies but does not state.

### KD6 — Foreign-producer fallback is to raw bytes, not to drop

A `obs decode` against a binary batch from a different service is a
common workflow. Dropping unknown schemas would make the CLI useless
in a multi-team workspace. The `payload_proto: bytes` column +
`{"_unknown_schema": true}` JSON renderer are the contract.
